# v0.2.0 MiroFish 预测评审框架

## 核心机制

**预测-实现对比**：每回先由 MiroFish 预测，再对比实际实现，分析差异

## 预测流程

### 第81回预测
- **输入**: 前80回《红楼梦》原文
- **输出**: 第81回预测（剧情、人物、场景）
- **对比**: 与v0.2.0实际第81回实现对比

### 第82回预测
- **输入**: 前81回（80回原文 + 1回续作）
- **输出**: 第82回预测
- **对比**: 与v0.2.0实际第82回实现对比

### 第N回预测（N≥82）
- **输入**: 前N-1回（80回原文 + N-81回续作）
- **输出**: 第N回预测
- **对比**: 与v0.2.0实际第N回实现对比

## 预测内容

### 每回预测包含
1. **剧情走向** - 本回核心事件
2. **人物命运** - 涉及人物及变化
3. **场景设计** - 关键场景描述
4. **判词呼应** - 本回落实的判词
5. **情感基调** - 悲喜、紧张度

### 预测格式
```markdown
## 第X回预测

### 输入
- 前文摘要: ...
- 关键线索: ...

### 预测输出
**剧情**: ...
**人物**: ...
**场景**: ...
**判词**: ...
**基调**: ...

### 置信度
- 整体: XX%
- 剧情: XX%
- 人物: XX%
```

## 对比分析

### 对比维度
1. **剧情一致性** - 预测vs实现的核心事件
2. **人物准确性** - 人物行为是否符合预测
3. **判词落实** - 判词是否被准确落实
4. **创新度** - 实现是否超越预测

### 分析格式
```markdown

### 实现回顾
```

## 评审标准

### 预测质量
- **高** (90%+) - 预测准确，实现符合预期
- **中** (70-89%) - 大致正确，有偏差
- **低** (<70%) - 预测偏离，需反思

### 实现质量
- **超越** - 实现优于预测
- **符合** - 实现符合预测
- **不足** - 实现未达预测

## 目标

通过 MiroFish 预测-实现对比，实现：
1. **判词落实准确率** 100%
2. **人物命运合理性** ≥95%
3. **剧情连贯性** ≥95%
4. **创新突破** 每回至少1处
