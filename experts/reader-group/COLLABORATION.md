## 二、协同流程

### 2.1 三层感知法

```
第一层：个人感知
    ├─ 每位金钗独立阅读
    ├─ 从本角色视角评价
    └─ 输出第一人称感受

第二层：群体对话
    ├─ 诗意组：黛玉、宝钗、湘云、妙玉 → 组内共识
    ├─ 治家组：凤姐、探春、李纨、平儿 → 组内共识
    ├─ 命运组：迎春、惜春、元春、可卿 → 组内共识
    └─ 边缘组：副册12钗 → 组内共识

第三层：综合共鸣
    ├─ 24钗联合声明
    ├─ 与曹雪芹对话
    └─ 与蒋勋美学对照
```

### 2.2 协作接口定义

**组内协作**:
```yaml
触发条件: 组内成员相关段落评审
参与成员: 组内全部成员
决策方式: 共识制（需全部认可）
输出: 组别共识报告
```

**跨组协作**:
```yaml
触发条件: 涉及多组人物场景
参与成员: 相关组别代表
决策方式: 协商制（2/3多数通过）
输出: 跨组共识报告
```

**核心层裁决**:
```yaml
触发条件: 组内/跨组无法达成共识
参与成员: CAO-001（曹雪芹）
决策方式: 最终裁决
输出: 裁决报告
```

### 3.2 群体进化

**诗意组进化会议**（每月一次）:
- 分享各自视角的新发现
- 讨论《红楼梦》原著新解读
- 优化组内协作流程

**全体金钗大会**（每季度一次）:
- 24钗联合声明更新
- 评审标准迭代
- SOULSPEC 规范升级讨论

- 新金钗加入时，学习历史评审记录
- 优秀评审案例入库
- 经验教训写入 MEMORY.md

## 四、冲突解决机制

### 4.1 冲突分级

| 级别 | 描述 | 解决方式 |
|------|------|----------|
| L1 | 单金钗内部犹豫 | 自我权衡 |
| L2 | 组内意见分歧 | 组内讨论，2/3多数 |
| L3 | 跨组观点冲突 | 核心层调解 |
| L4 | 原则性分歧 | CAO-001 最终裁决 |

### 4.2 裁决原则

1. **技术问题**（如判词落实）：以原著为准
2. **创意问题**（如场景设计）：提供多方案供用户选择
3. **版本问题**（如升级决策）：**必须提交用户裁决**
