# OpenClaw上下文控制策略

**日期**: 2026-03-16  
**问题**: 上下文超长导致模型性能下降

---

## 一、问题分析

### 上下文爆炸原因
1. **历史对话累积** - 多轮讨论后token数激增
2. **大文件内联** - 直接粘贴完整文档内容
3. **多任务并行** - 同时讨论多个回目

### OpenClaw限制
- 总上下文有上限（约32K-64K tokens）
- 超长时模型响应变慢/质量下降
- Session可能重置，丢失历史

---

## 二、解决方案

### 方案1：文件引用替代内联（推荐）

**❌ 错误做法**（内联完整内容）
```
用户：请修改以下章节：
[paste 5000字内容]
```

**✅ 正确做法**（引用文件路径）
```
用户：请修改 ~/apps/redream/05-Writing/02-Draft/ch081-draft.md
具体修改点：
1. 场景二增加贾环心理描写
2. 场景三黛玉梦境更朦胧
```

### 方案2：分段处理

**❌ 错误做法**（一次创作8回）
```
用户：创作81-88回初稿
```

**✅ 正确做法**（每回独立session）
```
用户：创作第81回初稿
...
[完成后]
...
用户：创作第82回初稿
```

### 方案3：精简提示词

**❌ 错误做法**（携带全部历史）
```
[前面20轮对话]
+ 新任务
```

**✅ 正确做法**（关键信息摘要）
```
[项目状态摘要]
- 大纲已确认
- 第81回创作中
- 关键决策：黛玉自缢...

新任务：创作第81回场景二
```

### 方案4：使用Subagent隔离

**推荐做法**
- 每个回目使用独立subagent
- subagent只接收必要上下文
- 主session只接收结果摘要

## 三、Redream项目应用

### 创作流程优化

| 阶段 | 原做法 | 优化后 |
|------|--------|--------|
| 大纲讨论 | 多轮对话累积 | 分文档保存，引用文件 |
| 初稿创作 | 一次创作8回 | 每回独立subagent |
| 评审反馈 | 全文粘贴 | 引用文件+具体行号 |
| 修改迭代 | 重复全文 | diff模式，只改指定部分 |

## 四、紧急压缩策略

如果上下文已超长：

### 方法1：重置Session + 续命提示词
```
[使用用户提供的续命提示词]
项目：Redream
当前：第X回创作中
关键决策：[摘要]
待办：[摘要]
```

### 方法2：分段交付
- 不等待8回全部完成
- 每2-3回交付一次
- 用户确认后继续

### 方法3：异步创作
- Subagent后台创作
- 主session定期poll状态
- 不阻塞等待
