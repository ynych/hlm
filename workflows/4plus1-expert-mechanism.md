## 一、4+1机制定义

### 1.1 标准流程

```
Step 1: 从专家库中随机选择4位专家
        ↓
Step 2: 分析这4位专家的领域覆盖
        ↓
Step 3: 生成第5位专家（+1），要求：
        - 与现有4位差异最大
        - 能填补领域空白
        - 具有互补性
        - 有现实职业/人设对应
        ↓
Step 4: 将+1专家加入专家库（永久）
        ↓
Step 5: 5位专家共同完成评审
```

### 1.2 +1专家生成原则

| 维度 | 要求 | 说明 |
|:-----|:-----|:-----|
| **领域互补** | 填补4位专家未覆盖的维度 | 如已有结构/人物/历史，+1可选影视改编 |
| **方法论差异** | 不同的评审方法论 | 如已有学术派，+1可选创作派 |
| **现实人设** | 对应现实中可识别的职业 | 如"前Netflix内容总监"、"小红书红楼博主" |
| **能力证明** | 有能力完成评审任务 | 有明确的专业背景和评审标准 |

## 三、v0.1.5评审：4+1专家生成示例

### 3.1 随机选择4位专家

假设随机选出：
1. RED-001（红学文献）- 学术/传统
2. GEN-001（性别研究）- 学术/现代理论
3. ARC-001（结构叙事）- 创作/传统
4. ECO-001（经济逻辑）- 学术/历史

### 3.2 分析领域覆盖

| 已覆盖 | 未覆盖 |
|:-------|:-------|
| 学术视角（3/4） | 市场/商业视角 |
| 传统维度（2/4） | 新媒体/网络视角 |
| 文本内部（4/4） | 读者接受度 |
| 严肃批评（4/4） | 通俗传播 |

### 3.3 生成+1专家：MKT-001 市场接受度专家

**专家ID**: MKT-001  
**领域**: 市场接受度 / 读者反馈分析  
**权重**: 5%（v0.1.5起）  
**人设**: 前阅文集团高级策划 + 豆瓣阅读主编  
**加入版本**: v0.1.5（作为+1生成）  

**评审维度**:
- 目标读者群体匹配度（40%）
- 开篇吸引力/留存率预估（30%）
- 付费转化潜力（30%）

**互补性**:
- 与前4位学术/创作专家形成"学院派vs市场派"互补
- 填补"读者接受度"空白维度
- 现实人设：阅文+豆瓣背景，有实际市场数据支撑

## 六、机制优势

1. **动态扩充**: 每次评审生成1位新专家，专家库持续进化
2. **领域覆盖**: 确保每次评审都有互补视角
3. **现实锚定**: 专家都有现实职业/人设，评审标准可验证
4. **避免固化**: 随机选择+动态生成，避免专家固化思维
