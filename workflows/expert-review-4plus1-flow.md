## 二、评审流程（5天周期）

### Day 1: 任务分配

**输入**: 
- 评审对象全文（vX.Y.Z）
- 评审方案（含检查清单）
- 上一轮评审汇总（如有）

**动作**:
1. 向5位专家发送评审邀请
2. 分配评审文档模板
3. 明确交付截止时间（Day 3 18:00）

**输出**: 
- 5份评审任务书

### Day 2-3: 独立评审

**专家任务**:
1. 阅读全文（预估4-6小时）
2. 按检查清单逐项检查
3. 填写评审报告模板
4. 提交评分（百分制）+ 结论

**评审报告模板**:
```markdown
## 专家评审报告：[专家ID]

### 基本信息
- **评审领域**: [领域名称]
- **评审权重**: [X%]
- **评审日期**: YYYY-MM-DD

### 综合评分: [XX]/100

| 维度 | 分值 | 得分 | 说明 |
|:-----|:----:|:----:|:-----|
| [维度1] | XX | XX | [简述] |
| ... | ... | ... | ... |

### 三条核心优势
1. [优势1]
2. [优势2]
3. [优势3]

### 三条关键问题
| 问题 | 级别 | 位置 | 说明 | 建议 |
|:-----|:----:|:----:|:-----|:-----|
| [问题1] | P0/P1/P2 | 回目 | [简述] | [建议] |
| ... | ... | ... | ... | ... |

### 总体结论
□ 强烈推荐（95+）/ □ 推荐（85-94）/ □ 有条件推荐（75-84）/ □ 不推荐（<75）

**结论说明**: [一句话总结]
```

### Day 4: 汇总分析

**输入**: 5份专家评审报告

**动作**:
1. 计算加权综合评分
2. 汇总问题清单（按严重性排序）
3. 与上一轮评审对比差异
4. 生成评审汇总报告

**输出**:
```markdown
## 4+1专家评审汇总

### 综合评分
| 专家 | 领域 | 权重 | 评分 | 结论 |
|:-----|:-----|:----:|:----:|:-----|
| [专家1] | [领域] | X% | XX | [结论] |
| ... | ... | ... | ... | ... |
| **综合** | - | **80%** | **XX/100** | **X/5推荐** |

### P0/P1/P2问题汇总
| 级别 | 数量 | 问题清单 |
|:----:|:----:|:---------|
| P0 | X | [问题1] [问题2] ... |
| P1 | X | [问题1] [问题2] ... |
| P2 | X | [问题1] [问题2] ... |

### 与上一轮对比
| 指标 | 上一轮 | 本轮 | 变化 |
|:-----|:-------|:-----|:----:|
| 综合评分 | XX | XX | ±X |
| 推荐率 | X/X | X/X | ±X |
```

### Day 5: 修复规划

**输入**: 评审汇总报告

**动作**:
1. 确定P0/P1/P2修复清单
2. 估算修复工时
3. 分配修复任务
4. 制定修复计划

**输出**:
- `FIX_PLAN_vX.Y.Z.md`

## 三、专家调度规则

### 3.1 专家池管理

```
/root/apps/redream/experts/
├── available/          # 可调度专家
│   ├── ARC-001-profile.md
│   ├── ARC-002-profile.md
│   └── ...
├── busy/               # 忙中专家
│   └── ...
└── archive/            # 归档专家
    └── ...
```

### 3.2 调度优先级

1. **领域匹配度**: 优先选择领域完全匹配的专家
2. **历史参与度**: 优先选择参与过上一轮的专家（保持连续性）
3. **可用性**: 优先选择档期空闲的专家
4. **多样性**: 确保不同视角（如同时有CHR-001和CHR-002）

### 3.3 回避原则

- 同一轮评审中，避免专家之间存在利益冲突
- 避免专家与评审对象存在直接关联
- 避免同一专家连续参与超过3轮（防止疲劳）

## 四、质量检查点

### 4.1 评审前检查

- [ ] 全文版本号正确
- [ ] 检查清单已更新
- [ ] 5位专家已确认参与
- [ ] 评审模板已分发

### 4.2 评审中检查

- [ ] 专家按时提交报告
- [ ] 报告格式符合模板
- [ ] 评分与结论一致
- [ ] P0问题有明确位置和建议

### 4.3 评审后检查

- [ ] 综合评分计算正确
- [ ] 问题汇总无遗漏
- [ ] 修复计划可执行
- [ ] 评审文档已归档
