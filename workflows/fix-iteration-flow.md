# 修复迭代流程 (Fix Iteration Flow)

**版本**: v1.0  
**适用范围**: Redream项目P0/P1/P2问题修复  
**创建时间**: 2026-03-23  

---

## 一、修复分级定义

| 级别 | 定义 | 修复时限 | 验收标准 |
|:----:|:-----|:---------|:---------|
| **P0** | 阻断性问题，必须修复 | 24-48小时 | 100%通过复评 |
| **P1** | 重要问题，强烈建议修复 | 3-5天 | 90%以上专家认可 |
| **P2** | 优化问题，可选修复 | 1-2周 | 不降低评分即可 |

---

## 二、修复流程图

```
评审汇总报告
    ↓
提取P0/P1/P2问题
    ↓
制定修复计划
    ├─→ P0: 立即修复
    ├─→ P1: 本轮修复  
    └─→ P2:  backlog
    ↓
执行修复
    ↓
修复文档化
    ↓
复评验证（P0/P1）
    ↓
整合入全文
    ↓
版本更新
```

## 四、修复执行SOP

### Step 1: 创建修复文档（修复前）

**命名规范**:
```
fix-[回目]-[关键词].md
```

**示例**:
- `fix-094-miaoyu.md` (94回妙玉场景)
- `fix-089-baoyu.md` (89回宝玉罪名)

**文档内容**:
1. 修复目标
2. 原文问题（摘录原文+问题说明）
3. 修复方案（详细修改内容）
4. 修改说明（为什么这样改）
5. 与判词/前文的呼应

### Step 2: 执行修复（修复中）

**原则**:
- 最小改动原则：只改需要改的地方
- 风格一致原则：保持与原文语言风格一致
- 判词呼应原则：修改需与人物判词呼应

**检查点**:
- [ ] 修复内容已写入fix文档
- [ ] 修改前后对比清晰
- [ ] 未引入新问题

### Step 3: 复评验证（修复后）

**P0复评**:
- 由原专家（或同领域专家）快速确认
- 确认标准：问题已消除
- 时限：24小时内

**P1复评**:
- 由项目负责人或指定专家确认
- 确认标准：90%以上认可
- 时限：48小时内

**P2复评**:
- 可选，下一轮评审时一并检查
- 确认标准：不降低评分

### Step 4: 整合入全文

**检查清单**:
- [ ] 修复内容已合并入全文
- [ ] 版本号已更新（全文内所有vX.Y.Z）
- [ ] 软链已更新（全文.md → 新版本）
- [ ] STATUS.md已更新
