# 规则迭代流程 (Rule Iteration Flow)

**版本**: v1.0  
**适用范围**: Redream项目专家评审规则迭代  
**创建时间**: 2026-03-23  

---

## 一、流程目标

将每轮专家评审发现的新规则、新标准、新问题类型，固化为可复用的检查清单，供下一轮评审引用。

---

## 二、流程图

```
专家评审完成
    ↓
提取新规则/新标准
    ↓
规则分类（P0/P1/P2/建议）
    ↓
规则文档化
    ↓
归档至rules/目录
    ↓
更新主检查清单
    ↓
下一轮评审引用
```

---

## 三、规则类型定义

| 类型 | 代码 | 定义 | 示例 |
|:-----|:----:|:-----|:-----|
| 阻断性规则 | P0 | 必须修复，否则无法通过评审 | 商业敏感内容、历史事实错误 |
| 重要规则 | P1 | 强烈建议修复，影响评分 | 节奏问题、心理描写单薄 |
| 优化规则 | P2 | 可选修复，锦上添花 | 意象密度、对话微调 |
| 建议性规则 | SUG | 不强制，供参考 | 未来改编建议 |

## 六、规则迭代SOP

### Step 1: 提取新规则（评审完成后24小时内）

从专家报告中提取：
- 专家明确指出的"必须"、"应该"、"建议"
- 专家检查清单中的新增项
- 专家评分说明中的扣分点

### Step 2: 规则分类（自动+人工）

自动分类规则：
- 关键词"必须/强制/阻断" → P0
- 关键词"应该/强烈建议" → P1
- 关键词"可以/建议" → P2
- 关键词"未来/如果" → SUG

人工复核：由项目负责人确认分类

### Step 3: 规则文档化（48小时内）

按模板创建规则文档，命名格式：
```
RULE-[类型]-[序号]-[简短描述].md
```

### Step 4: 更新主检查清单

将新规则编号追加至：`deliverables/vX.Y.Z/review/CHECKLIST.md`

### Step 5: 评审引用

下一轮评审开始时，将更新的检查清单作为评审输入：
```
请依据以下检查清单进行评审：
- [ ] RULE-P0-001: [规则简述]
- [ ] RULE-P0-002: [规则简述]
...
```
