## 2. 交付物质量标准

### 2.1 大纲质量标准

#### 完整性
- [ ] 28回大纲完整
- [ ] 每回包含: 标题、情节摘要、关键场景、判词落实
- [ ] 五部结构清晰（渐衰→崩塌→离散→流浪→情榜）
- [ ] 人物命运与判词一一对应

#### 质量
- [ ] 六维度评审≥90分
- [ ] 判词100%落实
- [ ] P0问题=0
- [ ] P1问题≤3

#### 可追溯
- [ ] 评审报告完整
- [ ] 评审专家身份明确
- [ ] 问题分级记录完整

### 2.2 全文质量标准

## 3. 达标验证流程

### 3.1 自检验证
交付方提交前自检:
- 对照质量标准逐项检查
- 填写自检清单
- 标记问题项

### 3.2 专家评审
- 按专家评审规则执行
- 生成评审报告
- 问题分级

### 3.3 达标判定
```
IF 评审分≥达标线 AND P0=0 THEN
    状态 = 达标
ELSE IF 评审分≥达标线 AND P0=0 AND P1≤3 THEN
    状态 = 有条件达标（需修复P1）
ELSE
    状态 = 不达标（需修复）
```

### 3.4 锁定机制
- 达标交付物锁定
- 版本号标记
- 变更需走规则迭代流程

## 4. 问题分级与处理

### 4.1 P0级（致命）
- **定义**: 影响根本质量
- **处理**: 必须修复，修复后复验
- **示例**: 判词未落实、人物崩坏、历史错误

### 4.2 P1级（重要）
- **定义**: 影响质量
- **处理**: 建议修复，80%+修复率
- **示例**: 节奏问题、轻微OOC

### 4.3 P2级（优化）
- **定义**: 提升质量
- **处理**: 可选修复
- **示例**: 文学性提升
