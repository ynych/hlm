## 一、项目相关术语

### GUIYU / 癸酉本

**英文**: Guiyu Edition / Wu's Stone Record  
**全称**: 《癸酉本石头记》（又称《吴氏石头记》）  
**说明**: 2008年起在网络上流传的《红楼梦》108回抄本，自称是真本后28回

**在本项目中**:
- 指代我们续写的基础架构
- 保留其情节框架，但润色文笔
- 弱化政治隐喻，强化文学性

---

### GUIYU-SEQUEL / 癸酉续红

**英文**: Guiyu Sequel  
**全称**: 癸酉续红——基于癸酉本架构续写《红楼梦》第81-108回  
**说明**: 本项目的正式名称

**特点**:
- 基于癸酉本的情节骨架
- 重写为符合曹雪芹文风的文学版本
- 共28回（第81-108回）

## 二、Agent相关术语

### Agent

**英文**: Agent  
**中文**: 智能体  
**说明**: 具有特定职责的AI助手，负责项目中的一部分工作

**本项目中的Agent**:

| Agent | 全称 | 职责 |
|-------|------|------|
| PM-Agent | Project Manager Agent | 项目经理，统筹进度 |
| ARC-Agent | Architecture Agent | 架构师，设计情节架构 |

---

### OOC

**英文**: Out Of Character  
**中文**: 人物性格偏离  
**说明**: 人物行为与前80回设定不符

**示例**:
- 黛玉变得世故圆滑 → OOC
- 宝钗主动追求宝玉 → OOC
- 宝玉热衷于科举 → OOC

**处理**: CHR-Agent审核，发现OOC后退回修改

## 三、《红楼梦》相关术语

### 12钗 / 金陵十二钗

**英文**: Twelve Beauties of Jinling  
**说明**: 《红楼梦》中的12位主要女性角色

**正册12钗**:
1. 林黛玉 - 绛珠仙草，泪尽而逝
2. 薛宝钗 - 金玉良缘，终是空
3. 贾元春 - 贤德妃，虎兕相逢
4. 贾探春 - 远嫁海疆
5. 史湘云 - 白首双星
6. 妙玉 - 欲洁何曾洁
7. 贾迎春 - 子系中山狼
8. 贾惜春 - 独卧青灯古佛旁
9. 王熙凤 - 机关算尽太聪明
10. 巧姐 - 偶因济刘氏
11. 李纨 - 晚韶华
12. 秦可卿 - 情天情海

---

### 判词

**英文**: Fate Verse / Prophecy Verse  
**说明**: 第五回中，宝玉在太虚幻境看到的预示人物命运的诗词

**示例**: 黛玉判词
```
可叹停机德，堪怜咏絮才。
玉带林中挂，金簪雪里埋。
```

**作用**: 前80回埋下伏笔，后28回应验

---

### 伏笔

**英文**: Foreshadowing  
**说明**: 前80回中暗示后续情节的细节

**类型**:
- 判词伏笔：12钗命运预言
- 诗词伏笔：灯谜、花签、酒令
- 对话伏笔：人物对话中的暗示
- 梦境伏笔：太虚幻境预言

---

### 癸酉

**英文**: Guiyou  
**说明**: 干支纪年，对应1693年或1753年

**争议**:
- 癸酉本声称完成于癸酉年（1693或1753）
- 学术界普遍认为是现代伪作
- 本项目只取其情节框架，不论真伪

## 四、写作相关术语

### 仿写

**英文**: Imitation Writing  
**说明**: 模仿特定作者文风进行创作

**本项目仿写对象**: 曹雪芹

**仿写要点**:
- 语言风格：京白与文言交融
- 叙事视角：全知视角，细腻心理
- 意象运用：花、竹、雪、月等
- 对话特点：个性化，符合人物身份

---

### 回末诗

**英文**: End-of-Chapter Poem  
**说明**: 每回结尾的总结性诗词，通常为七绝

**前80回示例**:
```
满纸荒唐言，一把辛酸泪。
都云作者痴，谁解其中味。
```

**本项目**: 每回必须包含回末诗，共28首

---

### 章回体

**英文**: Chapter-and-Verse Style  
**说明**: 中国古典小说的叙事形式

**特点**:
- 分回立目：每回有对仗的回目
- 说书人口吻："且说"、"话说"
- 悬念设置："欲知后事如何，且听下回分解"

**本项目**: 严格遵循章回体格式

---

## 五、项目阶段术语

### Phase

**英文**: Phase  
**中文**: 阶段  
**说明**: 项目的一个主要阶段

**本项目8个Phase**:

| Phase | 名称 | 对应目录 | 产出 |
|-------|------|---------|------|
| Phase 0 | 立项准备 | `01-Management` | 项目章程 |
| Phase 1 | 立意框架 | `02-Planning` | 人物档案 |
| Phase 2 | 大纲设计 | `03-Design/01-Phase2-Outline` | 28回大纲 |
| Phase 3 | 设计材料 | `03-Design/02-Phase3-Materials` | 伏笔/诗词/对话 |
| Phase 4 | 章节初稿 | `05-Writing/02-Draft` | 初稿 |
| Phase 5 | 初审 | `06-Quality` | 文风+人物审核 |
| Phase 6 | 二审统稿 | `06-Quality` | 修改稿 |
| Phase 7 | 终审交付 | `05-Writing/04-Final` | 定稿 |

---

### Workflow Model

**英文**: Workflow Model  
**中文**: 工作流模型  
**说明**: 本项目采用的架构模式

**与Phase-Based的区别**:
- Phase-Based：按时间阶段划分（Phase 0-7）
- Workflow-Based：按工作流程划分（管理→规划→设计→创作→质量）

**本项目**: 使用Workflow Model，目录结构按工作流组织
