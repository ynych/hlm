## 一、流程概览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   建立标准   │ → │  Agent评估   │ → │  输出报告   │ → │  用户验收   │
│  (用户确认)  │    │  (自动执行)  │    │  (问题清单)  │    │  (选择题)   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       ↑                                                    │
       └──────────────── 修复优化 ←────────────────────────┘
```

**用户角色**: 只做选择题（验收标准、确认规则、决策方向）  
**Agent角色**: 自动执行评估、输出报告、提出修复建议

## 四、自动化评估流程

### 4.1 评估触发条件

- **定时触发**: 每周一次自动评估
- **事件触发**: 大纲/人设更新后自动评估
- **手动触发**: 用户随时要求评估

### 4.2 Agent评估步骤

**Step 1: 自检**
- Agent读取当前所有文档
- 按照评估规则打分
- 列出问题清单

**Step 2: 交叉验证**
- 检查大纲与POV是否一致
- 检查不同角色POV是否矛盾
- 检查前80回与后28回衔接

**Step 3: 生成报告**
- 各模块得分
- 问题清单（P0/P1/P2分级）
- 修复建议

**Step 4: 提交用户**
- 报告呈现给用户
- 用户做选择题决策

### 4.3 输出报告模板

```markdown
# 专家评估报告 #{编号}
**日期**: {日期}
**评估范围**: {模块}

## 得分总览
| 模块 | 得分 | 目标 | 差距 |
|------|------|------|------|
| 主旨 | {分} | 90+ | {差距} |
| 设定 | {分} | 90+ | {差距} |
| 大纲 | {分} | 95+ | {差距} |
| 人设 | {分} | 90+ | {差距} |
| 剧情 | {分} | 95+ | {差距} |

## 问题清单
### P0级（必须修复）
1. {问题描述} → {修复建议}

### P1级（建议修复）
1. {问题描述} → {修复建议}

### P2级（可选优化）
1. {问题描述} → {修复建议}

## 用户决策点
- [ ] 问题1: A.修复 B.忽略 C.其他
- [ ] 问题2: A.修复 B.忽略 C.其他
```
