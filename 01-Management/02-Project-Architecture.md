# 项目架构设计

**架构模式**: 工作流模型（Workflow Model）  
**设计理念**: 按创作流程组织，从规划到交付  
**创建日期**: 2026-03-15  
**版本**: v1.0

---

## 一、架构概览

本项目采用**工作流模型**，按创作流程划分为7个阶段：

```
管理 (01-Management)
    ↓
规划 (02-Planning)
    ↓
设计 (03-Design)
    ↓
智能体 (04-Agents) → 协作
    ↓
创作 (05-Writing)
    ↓
质量 (06-Quality)
    ↓
文档 (07-Docs)
```

---

## 二、工作流阶段详解

### Stage 1: 管理 (01-Management)

**职责**: 项目管理、决策记录、架构设计

**核心文件**:
- `00-Project-Navigation.md` - 项目入口导航
- `01-Project-Overview.md` - 项目总体规划
- `02-Architecture.md` - 项目架构设计（本文档）
- `03-Roles.md` - Agent角色定义
- `04-Decisions/` - 决策记录
- `05-Status.md` - 项目状态看板
- `06-Risks.md` - 风险跟踪
- `07-Tech-Stack.md` - 技术栈
- `08-Git-Workflow.md` - Git工作流
- `09-Glossary.md` - 术语表
- `10-FAQ.md` - 常见问题

**交付物**: 项目章程、架构文档、决策记录

---

### Stage 2: 规划 (02-Planning)

**职责**: 调研、人物设计、参考资料整理

**子目录**:
- `01-Framework/` - 核心立意与框架设计
- `02-Characters/` - 人物档案（金陵十二钗等）
- `03-Reference/` - 参考资料、词汇表
- `04-Planning-Decisions.md` - 规划阶段决策

**交付物**: 人物档案、参考材料、框架设计

**输入**: 癸酉本研究、前80回分析  
**输出**: 完整的人物档案和参考资料

---

### Stage 3: 设计 (03-Design)

**职责**: 大纲设计、关键场景、设计材料

**子目录**:
- `01-Phase2-Outline/` - 28回详细大纲
  - `00-Phase2-Plan.md` - 大纲阶段规划
  - `Outline_Confirmation_Checklist.md` - 大纲确认清单
  - `关键剧情导航.md` - 快速跳转导航
  - `关键剧情对照表.md` - 前80回对照
  - `outline-XXX-v2.md` - 各回大纲（28个）
  - `motivation-review-XXX.md` - 动机审核（28个）
- `02-Phase3-Materials/` - Phase 3设计材料
  - `foreshadowing-table-v2.md` - 伏笔照应表
  - `poetry-collection-v1.md` - 诗词集
  - `key-dialogues-v1.md` - 关键对话设计
- `03-Key-Scenes/` - 重点场景设计

**交付物**: 28回大纲、伏笔表、诗词、对话设计

**输入**: 人物档案、癸酉本架构  
**输出**: 完整的设计文档，可直接进入创作

---

### Stage 4: 智能体 (04-Agents)

**职责**: 多Agent协作系统

**子目录**:
- `01-PM/` - PM-Agent（项目经理）
- `02-ARC/` - ARC-Agent（架构师）
- `03-CHR/` - CHR-Agent（人物审核）
- `04-STY/` - STY-Agent（文风审核）
- `05-WRT/` - WRT-Agent（写作执行）
- `06-EDT/` - EDT-Agent（统稿编辑）
- `07-REV/` - REV-Agents（外部评审）

**协作流程**:
```
PM分配任务 → ARC设计 → WRT写作 → CHR审核 → STY审核 → EDT统稿 → REV评审
```

**交付物**: 各Agent的工作成果和审核报告

---

### Stage 5: 创作 (05-Writing)

**职责**: 章节创作（初稿→修订→定稿）

**子目录**:
- `00-Style-Guide.md` - 文风仿写指南
- `01-Start-Guide.md` - 快速开始指南
- `02-Draft/` - 初稿（Phase 4）
  - `ch081-draft.md` 至 `ch108-draft.md`
- `03-Revision/` - 修订稿（Phase 5-6）
  - `ch081-rev1.md` 至 `ch108-revN.md`
- `04-Final/` - 定稿（Phase 7）
  - `ch081-final.md` 至 `ch108-final.md`

**交付物**: 28回完整章节（初稿、修订稿、定稿）

**输入**: 大纲设计、诗词、对话  
**输出**: 符合文风指南的完整章节

---

### Stage 6: 质量 (06-Quality)

**职责**: 质量保障、反馈收集

**子目录**:
- `01-Feedback/` - 用户反馈
  - `user-feedback-XXX.md`
- `02-Quality-Reports/` - 质量报告
  - `quality-report-phase5.md`
  - `quality-report-phase6.md`

**交付物**: 质量报告、反馈汇总

---

### Stage 7: 文档 (07-Docs)

**职责**: 文档资料、研究归档

**子目录**:
- `00-Architecture-Redesign-Proposals.md` - 架构设计方案
- `01-Cao-Analysis/` - 曹雪芹研究
- `02-Guixu-Analysis/` - 癸酉本研究
- `03-History/` - 历史背景

**交付物**: 研究资料、分析文档

## 四、命名规范

### 目录命名
- 使用英文编号前缀：`01-`, `02-`, `03-`...
- 使用英文描述：`Management`, `Planning`, `Design`...
- 示例：`01-Management`, `03-Design`, `04-Agents`

### 文件命名
- 使用英文（kebab-case或CamelCase）
- 编号前缀用于排序：`00-`, `01-`, `02-`...
- 示例：
  - `00-Project-Navigation.md`
  - `Outline_Confirmation_Checklist.md`
  - `foreshadowing-table-v2.md`

### 内容语言
- **所有内容使用简体中文**
- 代码/脚本使用英文
- 专有名词保留（GitHub, Markdown等）

## 六、与旧架构对比

### 旧架构（Phase-Based）
```
deliverables/phase1/ - Phase 1 交付物
deliverables/phase2/ - Phase 2 交付物
deliverables/phase3/ - Phase 3 交付物
agents/ - Agent系统
characters/ - 人物档案
```

### 新架构（Workflow-Based）
```
01-Management/ - 项目管理（新增）
02-Planning/ - 规划（原characters + reference）
03-Design/ - 设计（原deliverables/phase2-3）
04-Agents/ - 智能体（原agents）
05-Writing/ - 创作（原chapters）
06-Quality/ - 质量（原feedback）
07-Docs/ - 文档（原docs）
```

**优势**:
- 更符合实际工作流程
- 阶段职责更清晰
- 便于跟踪进度
- 易于新成员理解
