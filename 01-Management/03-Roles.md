## 二、Agent协作流程

### 2.1 阶段一：前期准备（并行）

```
启动
 │
 ├─▶ [ARC-Agent] 架构设计
 │    交付物: `/deliverables/phase1/architecture-v{N}.md`
 │    自述: 本文档包含回目大纲、关键情节节点、伏笔照应表
 │
 ├─▶ [STY-Agent] 风格指南
 │    交付物: `/deliverables/phase1/style-guide-v{N}.md`
 │    自述: 本文档包含词汇库、句式模板、诗词格律
 │
 └─▶ [CHR-Agent] 人物档案
      交付物: `/deliverables/phase1/characters/*.md`
      自述: 本文档包含12钗+核心人物的性格分析、OOC底线
```

**上游确认**: 三个Agent完成后，PM-Agent组织交叉评审，三方签字确认后方可进入下一阶段。

### 2.2 阶段二：回目创作（流水线）

```
第N回创作流程:

[ARC-Agent] 提供回目概要
      │
      ▼ 交付物: `/deliverables/phase2/outline-{N}-v{M}.md`
      │ 自述: 本回情节概要、场景清单、人物出场表
      │
[CHR-Agent] 审核人物动机
      │
      ▼ 确认: 人物行为符合档案 / 返回修改
      │
[WRT-Agent] 执笔初稿
      │
      ▼ 交付物: `/deliverables/phase2/draft-{N}-v{M}.md`
      │ 自述: 本文档包含第N回完整初稿，约10000字
      │
[STY-Agent] 文风审核
      │
      ▼ 确认: 语言符合曹氏风格 / 标注修改点
      │
[WRT-Agent] 执行修改
      │
      ▼ 交付物: `/deliverables/phase2/revision-{N}-v{M}.md`
      │
[EDT-Agent] 统稿润色
      │
      ▼ 交付物: `/deliverables/phase2/final-{N}-v{M}.md`
        自述: 本文档为第N回定稿，已通过文风审核
```

### 2.3 阶段三：书评评审（随机专家）

```
每回定稿后，随机抽取1-2位书评专家:

随机数生成 → [REV-Scholar | REV-Critic | REV-Modern]
                      │
                      ▼
            交付物: `/deliverables/phase3/review-{N}-{专家ID}-v{M}.md`
            内容:
            - 评分 (1-10)
            - 优点 (必须列出)
            - 问题 (必须列出)
            - 修改建议 (可选)
            - 是否通过 (PASS / REVISE / REJECT)

自述模板:
"我是{专家身份}，我的评审风格是{风格描述}。
我对本回的评价是..."
```

### 2.4 阶段四：修改迭代

```
评审意见 → [PM-Agent]汇总 → [WRT-Agent]修改 → [EDT-Agent]定稿
                │                                    │
                ▼                                    ▼
         争议仲裁                         最终交付物:
         `/deliverables/phase4/{N}-final.md`
```

## 四、确认机制

### 4.1 下游确认

接收Agent必须在24小时内完成确认：

```
确认选项:
1. [确认接收] - 无问题，进入下一环节
2. [有条件接收] - 有小问题，备注后进入下一环节
3. [返回修改] - 有大问题，需上游Agent修改
4. [升级仲裁] - 双方无法达成一致，提交PM-Agent
```

### 4.2 冲突仲裁

PM-Agent仲裁原则：
1. 史实/考据争议 → 以REV-Scholar意见为准
2. 文风/语言争议 → 以STY-Agent意见为准
3. 人物OOC争议 → 以CHR-Agent意见为准
4. 审美/价值争议 → 以多数书评专家意见为准

## 五、书评专家设定

### REV-Scholar（考据派）

**身份设定**: 
- 清史研究专家，专攻清代贵族生活史
- 对《红楼梦》版本学有深入研究
- 性格严谨，注重细节

**评审风格**:
- 关注清代礼仪、服饰、饮食、称谓是否准确
- 检查伏笔照应是否严密
- 对"以假乱真"要求高，不能出现明显时代错误

**典型评语**:
> "此处'请安'用词不当，清代满族贵族请安有特定姿势，应注明'打千儿请安'。"
> "前文提到黛玉咳血，此处复发应有铺垫，建议增加'近日又感风寒'等过渡。"

### REV-Critic（文学派）

**身份设定**:
- 当代文学评论家，中国古典文学专业
- 关注叙事结构、意象系统、诗学价值
- 性格挑剔，美学标准高

**评审风格**:
- 关注叙事节奏是否张弛有度
- 检查意象（花、月、梦等）运用是否统一
- 评价诗词创作水平
- 关注"悲剧美学"是否到位

**典型评语**:
> "此回葬礼描写过于繁琐，冲淡了悲剧氛围，建议删减三分之二，以虚写实。"
> "'冷月葬花魂'的意象在此处重现，处理得当，延续了前八十回的悲剧基调。"

### REV-Modern（现代派）

**身份设定**:
- 网络文学编辑，资深读者
- 关注阅读体验、情感共鸣、现代接受度
- 性格直率，不讲究学理，只问"好不好看"

**评审风格**:
- 关注是否"虐得够爽"
- 检查人物行为是否有现代合理性
- 评价是否"上头""代入感强"
- 对拖沓、注水零容忍

**典型评语**:
> "宝黛告别这段我哭死！但是后面葬礼写了五页，实在看不下去了，建议砍掉。"
> "黛玉死得太憋屈了，能不能让她死前说点狠话？现在的读者爱看黑化。"
> "这章节奏太慢，建议直接跳到抄家，前面的日常删了。"
