## 二、高风险 🔴

### Risk-001: 大纲变更导致返工

| 属性 | 内容 |
|------|------|
| **风险ID** | Risk-001 |
| **风险描述** | 用户在大纲确认阶段提出重大修改，导致已完成的设计材料需要返工 |
| **可能性** | 中 (50%) |
| **影响程度** | 高 |
| **风险等级** | 🔴 高风险 |
| **状态** | 活跃 |
| **发现日期** | 2026-03-15 |

**详细说明**:
- 当前28回大纲已完成，但用户正在review
- 如果用户对宝钗人设或关键情节提出重大调整
- 可能导致大纲修改，进而影响已完成的伏笔表、诗词、对话设计

**缓解措施**:
1. ✅ 已将大纲拆分为28个独立文件，降低耦合
2. ✅ 已完成动机审核（CHR），确保人物一致性
3. ⏳ 等待用户逐项确认，减少大范围变更可能性
4. ⏳ 准备快速调整方案

**应急计划**:
- 如果变更范围小（1-3回）：直接修改相关文件
- 如果变更范围大（5回以上）：评估是否重启Phase 2-3

**负责人**: PM-Agent  
**跟踪频率**: 每日，直到大纲确认完成

## 七、风险响应流程

### 风险发现
1. 任何Agent发现风险时，立即报告PM-Agent
2. PM-Agent评估风险等级（高/中/低）
3. PM-Agent更新本文档

### 风险处理
- 🔴 高风险：立即召开紧急会议，制定应对方案
- 🟡 中风险：纳入周会讨论，监控进展
- 🟢 低风险：记录并定期review

### 风险关闭
当风险不再存在时，更新状态为"已解决"，记录解决日期和方法
